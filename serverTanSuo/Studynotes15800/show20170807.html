1、Java基础类库
a)与控制台互动
运行Java程序的参数
JVM调用main()方法
如何为main()方法传入实际参数
Java命令运行main方法所在的类的时候，在类名后紧跟一个或者多个字符串
字符串用空格隔开，JVM会把这些字符串依次赋值给args数组
如果字符串参数本身包含空格，则应该用""将该字符串包裹起来

使用Scanner获取键盘输入
基于正则表达式的文本扫描器，可以从输入流、接收的文件、字符串中解析出基本数据类型的值和字符串的值
Scanner类提供了多个构造器，不同的构造器可以接收文件、输入流、字符串作为数据源
Scanner主要提供了hasNextXxx()、nextXxx()两个方法来扫描输入
默认使用空白（空格、tab、回车）作为多个输入项之间的分隔附

b)系统相关类
System
代表当前Java程序的运行平台,程序不能创建System对象
运行平台配置、环境变量、系统属性的查看

RunTime
运行环境，JVM相关信息
如处理器数量、内存信息
Runtime类代表Java程序的运行时环境，每个Java程序都有一个与之对应的Runtime实例
Runtime类代表Java程序的运行时环境，可以访问JVM的相关信息，如处理器数量、内存信息等。
Runtime的exec方法可以单独启动一个进程来运行操作系统的命令
rt.exec(“notepad.exe”);

Object
equals方法比较对象的地址是否相等(通常情况下会重写equals方法自定义相等的比较规则，object的equals没有实际意义)
native 本地代码实现，一般为c、c++
Object类是所有类的父类，也就是说，Java允许把任何类型的对象赋值给Object类型的变量。
任何Java类的对象都可以调用Object类的方法。
equals(Object obj)
finalise()
getClass()
hashCode()
toString()

String
（String str1="...";方法区常量池）
（String str1=new String("...");堆）
String的值一旦确定不可更改，只能新建一个
==与equals的区别
a)split() 字符串分割
b)substring() 字符串截取 前包括、后不包括
c)toUpperCase() 全大写
d)toLowerCase() 全大写

StringBuffer
线程安全的可变字符序列

StringBuilder
线程不安全的可变字符序列，效率高于StringBuffer

Math
重点掌握随机数Random();

ThreadLocalRandom()和Random()的区别：
种子相同运算规则相同（伪随机）
推荐选择当前时间的毫秒值作为种子
前者线程安全，后者线程不安全

BigDecimal
两个基本类型的浮点数运算容易引起精度丢失。如：(0.05+0.01)、(1.0-0.42)
建议使用基于String的构造器
如果必须使用double类型作为构造器的参数，则通过BigDecimal.valueOf(double value)静态方法来创建BigDecimal对象
可以以BigDecimal为基础定义一个工具类进行运算

格式化日期，日期与字符串相互转换

代码示例：
  /*String->Date*/
  // 设置时间字符串
  String datestr = "2017-05-06";
  // 设置日期格式
  SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");
  // 把字符串日期变为date日期
  Date date = sdf.parse(datestr);

  /*Date->String*/
  // 设置日期格式
  sdf = new SimpleDateFormat("yyyy年MM月dd日");
  // 为Date设格式
  datestr = sdf.format(date);

  /* 第二种方式 */
  /*String->Date*/
  LocalDate localDate=LocalDate.parse("2017-05-06");
  DateTimeFormatter dtf=DateTimeFormatter.ofPattern("yyyy年MM月dd日");
  /*Date->String*/
  System.out.println(localDate.format(dtf));

2、Java 异常机制
a)异常的概述
try catch finally throw throws
异常处理已经成为衡量一门语言是否成熟的标准之一，除了传统的像C语言没有提供异常处理机制之外，目前主流的编程语言都提供了异常处理机制。
对于一个程序设计人员，要尽可能地预知所有可能发生的情况，尽可能地保证程序在所有糟糕的情形下都可以运行。
国内的程序员做开发时，往往只做了“对”的事
使用if语句错误处理机制，有如下缺点：
无法穷举所有的异常情况
错误处理代码和业务实现代码混杂

b)Java中异常处理机制
捕获并处理 try catch finally
跑出 throw throws

c)Checked异常和Runtime异常体系
Checked异常：编译异常，编译时必须处理1、try-catch；2、throws
Runtime异常：运行时异常，利用修改业务逻辑来避免，不建议处理

d)try-catch语句块
所有父类异常的catch块都应该排在子类异常的后面，先处理小异常，再处理大异常
如果try内的语句块被执行一次就有异常抛出，则try后的catch分支做多有一个分支内的语句会被执行

e)jdk1.7提供多异常捕获
异常类型用|隔开
异常变量是一个隐式的final修饰的常量

f)自定义异常(继承Exception)
自定义一个类继承Exception或者RuntimeException
在方法的指定位置跑出该异常对象throw
在方法签名后抛出异常类型throws
throw抛出异常对象，用在方法体中
throws抛出异常类型，用在方法签名后
catch和throw的配合使用

g)finally
用在try-catch后，不管抛不抛异常都会执行
通常用来回收try中打开的物理资源(数据库连接、网络连接、磁盘文件读写)
这些物理资源一定要显性回收
将资源回收的代码写入finally中，一定会被执行(除非调用exit终止虚拟机)
尽量避免在finally语句块中使用return和throw语句

h)Java的异常跟踪栈
getMessage()：返回该异常的详细描述字符串
printStackTrace():将该异常的跟踪栈信息输出到标准错误输出
printStackTrace(PrintStream s):将该异常的跟踪栈信息输出到指定输出流
getStackTrace():返回该异常的跟踪栈信息

i)异常处理的规则
不要过度使用异常
不要使用过于庞大的try块
避免使用catch all语句
不要忽略捕获到的异常

j)使用throw抛出异常
catch和throw配合使用
后台需要通过日志来记录异常发生的详细情况
应用还需要根据异常向应用使用者传达某种提示
