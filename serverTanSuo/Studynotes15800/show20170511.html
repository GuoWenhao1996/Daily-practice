Java有意思的三个方面：线程 通信 反射
-------------------接口------------------------
接口:面向接口编程 一旦接口的标准 ，我就必需按照接口去实现功能
接口 : 一种特殊的抽象类,里面定义常量 实际全局最终的常量 和 抽象 方法组成 接口的关键是interface
1.一个类可以实现多个接口 使用的关键字 是implements
2.一个类继承某个类还可以实现多个接口
3.接口 没有办法 去实例化 可以做为引用数据类型
4.接口与接口可以实现继承，并且可以多继承。
-------------------多态------------------------
多态 一个物体拥有多种形态
满足多态：
1.满足于继承或者是实现接口
2.子类重写父类的方法
3.父类的引用 指向 子类对象
--------------对象类型转换---------------------
前提：满足于继承或者实现某个接口
1. 向上转型 有子类转换父类
父类引用指向的子类对象
2. 向下转型 父类对象强制转换为子类对象 （尽量去避免）
不安全，可能会产生类型转换异常。
注意 ： 编译的看的是左边； 运行时看的是右边
-----------------object类----------------------
Object类 ：每个类都使用 Object 作为超类
equals 方法 比较对象是否相等
通过getClass（） 可以返回运行时的类
有wait 和 notify 等待 唤醒 涉及线程 详细说明
toString（） 返回此对象字符串表现形式
----------------instanceof---------------------

代码示例：
		Animal a = new Dog();
		Animal b = new Cat();
		// 判断当前运行对象是否属于某一个类型
		System.out.println(a instanceof Cat);// fasle
		System.out.println(a instanceof Dog);// true

---------------单例设计模式--------------------
单例 模式： 保证仅且只有唯一的一个实例 singleton
1.保证构造方法私有化
2.如何在本类去实例化自己
饿汉式：
public class Singleton {
    //2 创建一个私有的静态无法被修改的成员变量
	private static final Singleton  singleton  = new Singleton();
	//1.构造方法私有化
	private  Singleton(){

	}
	//3 提供一个静态方法来获取唯一的实例
	public  static   Singleton  getInstance(){
		return    singleton;
	}

}
懒汉式：
public class Singleton1 {

	private  static  Singleton1  singleton1;

	private  Singleton1(){

	}

	public synchronized static   Singleton1  getInstance(){
		  if(singleton1==null){
			  singleton1 = new Singleton1();
		  }
		  return   singleton1;
	}
}
总结：
保证当前只有一个线程调用这个方法
解决理论上会出现的同时调用这个方法，然后就不能保证单实例了
理论上的并发，数据的持久性唯一性
