MySQL数据库：
1、关系型数据库的概述
为什么使用数据库？
减少数据冗余度
提高数据的一致性、准确性、安全性
对数据进行统一的管理

数据库的发展经历了哪些阶段？
☆层次模型
☆网状模型
☆关系模型(实体、关系)

在关系型数据库中，数据是存储在表内的
表与表之间的关系
一对一
一对多
多对多 必须有一个中间表
表述关系 用外键
☆对象模型
no-sql

数据库、数据库管理系统、数据库系统，MySQL是什么定位？
计算机中按照一定规则集中存储数据的一个空间的抽象-->数据库(DB)
按照某种规则操作数据库中的数据-->数据库管理系统(DBMS RDBMS)
安装了数据库的计算机(硬件和软件的总称)-->数据库系统(DBS)

2、管理数据库和表
操作数据库：
☆展示MySQL所管理的数据库：
show databases;
show databases \g
show databases \G

☆创建数据库：
create database day01;

☆删除数据库：
drop database day01;

☆连接数据库(打开进入指定数据库)：
use day01;

操作表：
☆创建表：
create table studentinfo(
sid int,
sname varchar(20),
sex char(2),
age tinyint,
phonenum varchar(11)
);

☆查看数据库中的表：
show tables;//看所有表名
show create table studentinfo;//看建表语句
describe studentinfo;//看表结构
desc studentinfo;//看表结构简写

☆删除表：
drop table studentinfo;

☆向表中插入数据：
insert into studentinfo values(1,'张三','男',20,18812341234);

☆查询表中所有数据：
select * from studentinfo;

☆删除表中所有数据：
delete from studentinfo;

☆复制表：
create table studentcopy_1 as select * from studentinfo where 1=0;//只复制表结构
create table studentcopy_2 as select * from studentinfo;//复制表结构和内容

3、保证数据完整性
理解数据完整性定义
数据准确+可靠
需要在建表时规定字段的数据完整性约束

实现各种完整性的方法+语法
☆域完整性
数据类型

外键约束
foreign key (cid) references class(cid)
一张表中可以有多个外键，外键列一般关联到另一张表的主键列

非空约束
not null

默认值
default 值
注意数据类型
让默认值生效
insert into user(username,password) values ('admin',default);
insert into user(username) values ('admin1');

☆实体完整性
唯一约束 unique
主键约束 primary key 唯一,非空，索引列
注：同一个表中只能由一个主键
一个主键可以加在多个列上，由多个列组成的主键叫做复合主键
复合主键的每一列的值都不能为空
复合主键的值组合起来判断是否唯一
primary key(username,password)

标识列 auto_increment 整数的自动递增
注：可在建表时设置自增长的初始值

☆引用完整性
外键约束

☆自定义完整性
触发器...

最终的建表语句
create table user(
id int primary key auto_increment,
username varchar(20) unique not null,
password varchar(20) default '000000'
)auto_increment=1000;

修改表的结构
☆增加列：
alter table 表名 add column 列名 类型 约束;
例：alter table user add column status int default 1;

☆修改列:
修改数据类型、约束
alter table 表名 modify 列名 新数据类型 新约束;
例：alter table user modify username varchar(30) unique not null;

修改列名、数据类型、约束
alter table 表名 change 旧列名 新列名 新数据类型 新约束;
例：alter table user change username uname varchar(30) unique not null;

☆删除列
alter table 表名 drop 列名;
例：alter table user drop status;

☆修改主键、外键
alter table 表名 add [constraint 主键名] primary key (列1,列2);
alter table 表名 add [constraint 外键名] foreign key (列1) references 表(列);

数据的查询语句
增

日期和字符串需要使用单引号
如果一个字段的值没有输入项，则可以使用null或者default代替

删
delete from 表名 [where 条件];
truncate [table] 表名; （确切的说叫重置表，它不是DML语句，是DDL语句）

比较运算符
=

!=、<>
