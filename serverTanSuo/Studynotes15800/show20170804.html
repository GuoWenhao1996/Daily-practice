java初级学习-第5日：接口、抽象类、多态

0、补充
块
静态块：类加载时被调用
用来初始化静态成员变量的值
在程序加载时做一些初始化工作
实例化块
静态块>>>>实例化块>>>>构造方法
内部类
成员内部类
创建内部类对象必须先创建外部类对象，由外部类对象创建内部类对象
内部类能够直接调用外部类的方法
而外部类不能直接调用内部类的方法
静态内部类
public static class
可以直接创建对象
只能调用外部类静态的方法和静态成员变量
局部内部类
写在方法里的
不能加访问修饰符
只能在方法里用
匿名内部类
就是没有名字的内部类
常做方法的实际参数
通常用来简化代码的编写
必须继承一个父类或实现一个接口

1、信息类编写
属性私有化
提供访问器修改器
提供无参构造方法
2、接口
[public][abstract] interface 接口名称 [extends 接口1,接口2,...]{}
接口就是用来制定规范的
抽象的方法（抽象类）
接口的特点：
接口中的抽象方法都是public abstract 修饰的，即使省略了这两个单词，也是默认有这两个词的。
接口中的成员变量都是public static final 的公共静态常量
接口中可写静态方法 (jdk1.8)
默认方法用default修饰 (jdk1.8)
接口中可以写实例方法，必须用default修饰
一个接口可以继承多个接口，一个类可以实现多个接口

3、抽象类
使用abstract可选修饰符修饰类
不能被final修饰(final和abstract只能用一个)
抽象类和普通类的特点
抽象类不能创建对象，而普通类可以
抽象类里可能有抽象方法，而普通类里一定不能有抽象方法
抽象方法所在的类一定是抽象类
抽象类存在的目的就是为了被继承
抽象方法不能有方法体，要被子类重写

4、多态
同一种事物的不同展现形式
静态多态
在编译时确定
方法的重载
动态多态
只有在程序运行时确定
方法的重写
如何实现多态
类的继承或者接口的实现
方法的重写
父类的引用指向子类的对象
向上转型(上塑造性)
凡是父类类型的引用均可以接收子类型的实例(里氏替换原则)
向上转型后，子类新增的方法、属性都是不可见的
如果有方法的重写，调用的方法是子类重写后的方法，其他情
况都是基于父类 向上转型后调用的静态方法都是基于父类的
向下转型
Fruit f=new Banana();
Banana b=(Banana)f;

instanceof 左边是对象，右边是类型
Fruit f=new Banana();
Fruit f1=new Fruit();
f instanceof Banana --->true
f1 instanceof Banana --->false

多态的应用
异构集合(Fruit[] fruits=new Fruit();)
多态参数(方法需要父类类型，那么子类对象也可以传入)
方发表和方法的调用
