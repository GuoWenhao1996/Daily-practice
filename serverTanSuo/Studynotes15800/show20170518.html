--------------对自定义对象排序-----------------
Arrays 位置 java.util包
.sort 对指定数据进行升序的方式排序
1.第一种方式：
对自定义对象 必需 要实现 Comparable 接口 并且重写里面 compareTo 方法

	@Override
	public int compareTo(Student o) {
		if (this.getAge() > o.getAge()) {
			return 1;
		} else if (this.getAge() < o.getAge()) {
			return -1;
		} else {
			return 0;
		}
	}
简化写法
	@Override
	public int compareTo(Object o) {
		Student s = (Student) o;
		return s.age - this.age;
	}
通过 返回 0 1 -1 来决定顺序 比较： 2个之间比较

2.第二种方式： 策略模式基础上
对自定义对象建立不同的策略，也就建立不同策略对象 ，策略需要实现 Comparator 接口 重写 compare 的方法
体重策略
import java.util.Comparator;
import com.zr.model.Student;

/**
 * 自定义WeightStrage策略类，重写compare方法，按体重排序。
 *
 * @author guowh
 *
 */
public class WeightStrage implements Comparator{

	@Override
	public int compare(Object o1, Object o2) {
		Student s1 = (Student) o1;
		Student s2 = (Student) o2;
		return s1.getWeight() - s2.getWeight();
	}

}
//年龄策略
package com.zr.strage;

import java.util.Comparator;
import com.zr.model.Student;

/**
 * 自定义AgeStrage策略类，重写compare方法，按年龄排序。
 *
 * @author guowh
 *
 */
public class AgeStrage implements Comparator{

	@Override
	public int compare(Object o1, Object o2) {
		Student s1 = (Student) o1;
		Student s2 = (Student) o2;
		return s1.getAge() - s2.getAge();
	}
}
数组工具类为我们提供sort排序的方法 可以传入不同的策略 的到不同的结果
		/**
		 * 自定义数据类型排序
		 *
		 * 提供了在Student类里实现Comparable接口重写compareTo()方法
		 * 提供了AgeStrage策略类实现Comparator接口重写compare()方法
		 * 提供了WeightStrage策略类实现Comparator接口重写compare()方法
		 */
		Student s1 = new Student(11, 100);
		Student s2 = new Student(22, 150);
		Student s3 = new Student(5, 90);
		Student s4 = new Student(10, 200);

		Student[] students = new Student[] { s1, s2, s3, s4 };
		// error:Student cannot be cast to java.lang.Comparable
		//// 按 年龄 自定义排序（写在Student类里不好维护）
		// Arrays.sort(students);

		//// 按 年龄 自定义排序（添加AgeStrage策略类 好维护）
		Arrays.sort(students, new AgeStrage());

		//// 按 体重 自定义排序（添加WeightStrage策略类 好维护）
		Arrays.sort(students, new WeightStrage());
		for (Student student : students) {
			System.out.println(student.getAge() + "\t" + student.getWeight());
		}
--------------异常机制Exception----------------
任何程序都应该具有的 保证可移值性 ，安全，健壮
通过 try catch 来进行捕获异常 当然异常是可以捕获多个的 由小到大
		/**
		 * 异常捕获 从小到大
		 */
		// public static void main(String[] args) throws
		// FileNotFoundException
		// 不建议使用上面那种，建议下面这个
		try {
			InputStream in = new FileInputStream("/11/11/das");
		} catch (FileNotFoundException e1) {
			// TODO Auto-generated catch block
			e1.printStackTrace();
		}

		try {
			Integer.parseInt("aaaaa");
		} catch (NumberFormatException e) {
			e.printStackTrace();
			System.out.println(e);
			System.out.println(1000);
		} catch (Exception e) {
			e.printStackTrace();
			System.out.println(1111);
		} finally {
			System.out.println(2222);
		}
在使用finally的时候，建立捕获异常的时候，代码不管是否能发生异常，都会最终去执行finally里面的代码
final 和 finally 的区别
----------------异常的种类区别-----------------
异常是来自于 Throwable 下面分为了 2 个类 一个Error， Exception
Error 记录的系统产生的错误。（JVM）
Exception：（核心）
分为了2种类型的异常:
一种是 checkedException 检查异常 其它异常
一种是 运行异常 runtimeException
除开 runtimeException 其它的就是检查异常
checkedException： 相当于在编写代码的时候就必需要进行捕获
runtimeException： 运行时异常 ，可以选择捕获或者不捕获
-----------------throws和throw-----------------
throws 可以抛出多个异常 跟在我们方法的后面 可以定义抛出多个异常。
		public  void eat() throws FileNotFoundException,Exception{

		}
throw 只能是单个异常
          	public  void eat(int a) {
			  if(a==1){
				  throw  new RuntimeException();
			  }
			  System.out.println(1111);
		}
-------------------自定义异常------------------
（项目分得特别细的，自定义一套异常体系）
1 .继承Exception 或者 runtimeException
2.定义有参的构造方法
/**
 * 自定义电话号码异常类
 *
 * @author guowh
 *
 */
public class PhoneNumException extends Exception {
	public PhoneNumException(String msg) {
		super(msg);
	}
}
3.如何去使用
/**
 * 自定义UserService类，用于测试PhoneNumException
 *
 * @author guowh
 *
 */
public class UserService {
	private String phone;

	public String getPhone() {
		return phone;
	}

	public void setPhone(String phone) throws PhoneNumException {
		if ("1".equals(phone))
			throw new PhoneNumException("对不起！号码不能为 1");
		else if ("11".equals(phone))
			throw new PhoneNumException("对不起！号码不能为 11");
		else
			System.out.println("成功！");
	}
}
4.演示
		/**
		 * 自定义异常
		 */
		UserService us = new UserService();
		try {
			us.setPhone("111");
		} catch (PhoneNumException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
			System.out.println(e);
		}
