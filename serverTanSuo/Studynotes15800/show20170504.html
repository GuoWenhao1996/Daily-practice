-----------------string-------------------------
String类 自定义对象 引用类型 字符串
1.声明定义

代码示例：
		/*
		 * 两种声明方式
		 */
		String s = "ssss";
		String s1 = new String("ssssssssssssssssss");
		System.out.println(s);
		System.out.println(s1);
注意 ：在使用字符串的时候 会涉及到拼接 这个是使用 +
对象之间的比较 == 和 equals
== 是比较对象在内存的地址是否是一致
不使用new关键字来声明字符串，字符串放在内存字符串串池
如果再次声明，那么它会直接到串池去寻找
字符串的特性：字符串一旦声明，就无法随意改变字符串的内容，只会生成新的数据（新的地址）
代码示例：
		/*
		 * == 用来比较数据在内存中的地址是否一致
		 *
		 * string池（所有用""生成的字符串） 优先从串池中拿对象
		 */
		String s = "a";
		String s1 = new String("a");
		String s2 = new String("a");
		String s3 = "a";
		String s4 = "ab";
		String s5 = s + "b";
		System.out.println(s == s1); // false
		System.out.println(s1 == s2);// false
		System.out.println(s2 == s3);// false
		System.out.println(s == s3); // true
		System.out.println(s == s4); // false
		System.out.println(s4 == s5);// false
-----------------equals()和==-------------------
== 比较的是内存中的地址
在比较自定义对象 ，如果使用 equals方法，必需要重写equals方法（判断里面属性是否一样）
建议也同时重写hashcode方法
代码示例：
		/*
		 * equals() 与 ==
		 *
		 * 前者比较内容，后者比较地址
		 */
public class Student {
	private String sname;
	private int sage;

	public int getSage() {
		return sage;
	}

	public void setSage(int sage) {
		this.sage = sage;
	}

	public String getSname() {
		return sname;
	}

	public void setSname(String sname) {
		this.sname = sname;
	}

	public Student(String sname, int sage) {
		this.sname = sname;
		this.sage = sage;
	}

	/**
	 * 两个的对象相等
	 *
	 * @param s
	 *            传入的对象
	 * @return 相等true 不相等false
	 */
	public boolean equals(Student s) {
		return this.sname == s.getSname() && this.sage == s.getSage();

代码示例：
		/*
		 * API
		 */
		String s = "abc";
		// charAt();
		char c = s.charAt(1);
		System.out.println(c);
		// 自己写
		String s1 = "";
		for (int i = s.length() - 1; i >= 0; i--) {
			s1 += s.charAt(i);
		}
		System.out.println(s1.trim());
		// 调用
		StringBuffer s2 = new StringBuffer(s);
		System.out.println(s2.reverse());
		s2.append("d");
		System.out.println(s2);



		/*
		 * StringBuffer 效率高
		 */
		String s = "abc";
		String ss = "d";
		String sss = s + ss;
		System.out.println(s == sss);// fasle

		StringBuffer s1 = new StringBuffer(s);
		StringBuffer s2 = s1.append("d");
		System.out.println(s1 == s2);// true

继承 基于构造方法
--------------------继承-----------------------
继承 ： extends
public class Student extends Person {

}
继承只能是单继承： 父类只有一个 但是子类是可以有多个
继承： 子类继承父类 拿到了父类 属性 和 方法 私有的方法 私有的属性 子类没有办法使用
-----------------方法的重写--------------------
方法的重写 override
是建立在有继承的关系之上 才会涉及到方法的重写
子类重写父类的方法：
特性：
1. 子类在重写父类的方法的时候，包括方法名，返回值，形参列表 必需是完全一致的
2. 子类所抛出的异常不能超过父类所抛出的异常
3. 子类方法的访问级别不能低于父类访问级别
--------------------继承-----------------------
super 的关键字 指向的是父类的对象
通过使用super关键字：
衍生出 在构造方法中调用构造方法时候，只能写在当前构造方法的第一行，并且只能调用一次。 规矩 this（） super（）

-----------------final的关键字-----------------

-------------------抽象类----------------------
抽象类 abstract class ： 进一步进行了升华 进一步的抽象
person
student teacher dog -------- Animal
抽象类：
1. 只要包含抽象方法的 一定是抽象类
2. 修饰抽象类和抽象方法必然使用abstract关键字
3. 定义抽象方法，必然没有具体的实现。
4. 子类如果继承父类，子类必须要重写父类的抽象方法
5. 抽象类无法被实例化 ，但是可以做为引用数据类型
注意： 抽象类同样可以有具体实现的方法
