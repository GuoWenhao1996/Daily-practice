		int arg3 = arg1 + arg2;
		return arg3;

	// 方法3 形参使用 ... 可以传入多个参数

	}
}
什么是业务主键？
业务主键（自然主键）：在数据库表中把具有业务逻辑含义的字段作为主键，称为“自然主键(Natural Key)”。
逻辑主键（代理主键）：在数据库表中采用一个与当前表中逻辑信息无关的字段作为其主键，称为“代理主键”。
复合主键（联合主键）：通过两个或者多个字段的组合作为主键。

需求分析清楚后再写业务逻辑，少走弯路。

开发中无论做什么事情都要有返回值，反馈一下到底做了没有，做了什么。

值传递和引用传递：
值传递：当数据做为基础类型和String 在进行传递的时候，其本身的值是不会发生变化的。形参在这里只是接受这个值一份拷贝而已
代码示例：
public class Person {
	public int changAge(int age) {
		age++;
		return age;
	}
}
public class Test {
	public static void main(String[] args) {
		Person p = new Person();
		int age = 15; // 局部变量
		p.changAge(age);
		System.out.println(age);
	}
}
引用传递： 当数据作为自定义对象在进行传递，整个对象不会进行拷贝，直接由形参所接受，如果有方法操作了这个对象，理所应当的对象就会发生变化
代码示例：
public class People {
	public int age;

	public int changeAge(People p) {
		int age = p.age++;
		return age;
	}
}
注意：2个形参数接受的类型

方法重载：方法重载是指在一个类中定义多个同名的方法，但要求每个方法具有不同的参数的类型或参数的个数。
一.方法名一定要相同。
二.方法的参数必须不同，包括参数的类型或个数，以此区分不同的方法体。
1.如果参数个数不同，就不管它的参数类型了！
2.如果参数个数相同，那么参数的类型必须不同。
直白的说 就是方法的重载 着重关注与形式参数 方法名必须一致
代码示例：

	 * 更改年龄 通过年龄 和 名字

	 * @param age
	 *            年龄

成员变量 工作区域

	public static int a = 10;
	public int b = 10;

	public void a() {
		a++;
	}

	public void b() {
		b++;
	}
	public static void main(String[] args) {
		Student s = new Student();
		s.a();
		s.b();
		System.out.println(s.a);
		System.out.println(s.b);
		Student s1 = new Student();
		s1.a();
		s1.b();
		System.out.println(s1.a);
		System.out.println(s1.b);
	}
互相调用：
普通的方法 是属于对象的
静态的方法 是属于类的

普通方法之间可以互相调用,同样也可以直接调用静态方法
静态方法之间可以互相调用，但是如果需要调用普通方法，就必需要实例化对象，在使用点语法调用

递归：一个方法不断地重复一个行为（自己的行为）
直白的说就是自身调用自身
要在合适的时候添加终止条件
实际开发中尽量避免递归
要考虑在什么时候使用
代码示例：
	// 递归形式
	public static void digui() {
		digui();
	}
使用:尽量避免无限自身调用，
如果我要停止递归无限调用自己本事 需要给一个临界条件（中止信号）
代码示例：
	// 求单个阶乘
	public static void noUseDigui(int arg) {
		int arg1 = 1;
		for (int i = 1; i <= arg; i++) {
			arg1 = arg1 * i;
		}
		System.out.println(arg1);
	}

	// 求单个阶乘
	public static int UseDigui(int arg) {
		if (arg == 1) {
			return 1;
		}
		return UseDigui(arg - 1) * arg;
	}

	public static void main(String[] args) {
		Test3.noUseDigui(5);
		System.out.println(Test3.UseDigui(5));
	}

内部类
内部类本来是封闭的但是可以看到外面的东西
1.内部的类是可以方便的访问到外部类的属性
2.但是内部类破坏类一个完整的结构的
代码示例：
public class OutClass {
	private int a = 1;

	// 内部类
	class intClass {
		public void abc() {
			System.out.println(a);
		}
	}

	public static void main(String[] args) {
		// 需要实例化内部类的时候，首先实例化外部类
		OutClass out = new OutClass();
		// 实例化内部类
		OutClass.intClass in = out.new intClass();
	}
}
普通代码块·静态代码块·非静态代码块
代码示例：
public class Person {
	// 静态代码块
	static {
		System.out.println("aaaa");
	}
	// 非静态代码块
	{
		System.out.println("bbbb");
	}
	// 普通代码块
	public Person() {
		System.out.println("cccc");
	}
}
执行顺序：
静态代码块》 非静态代码块 》普通代码块
静态代码块 只在内存执行一次
